과제1 – 설계과제
주제: 상담예약 시스템 설계
과제 설명
- 우선 과제 전 과정에 대해서 AI 도구 사용이 가능함을 안내 드립니다.
- 상담예약 시스템 (상담사 관리 + 신청자 예약)을 설계합니다.
- 기본 프로세스:
① 상담사가 스케줄 설정 → ② 이메일 링크 발송 → ③ 신청자 예약 신청 → ④ 상담 진행 및 이력 관리
- 동시간대 최대 3명 예약 가능, 기본적인 동시성 처리가 필요합니다.
- 유지보수 가능한 기본 구조 설계를 해야합니다.
- 상담사(관리자)가 상담스케줄을 등록/수정/삭제 할 수 있어야 하며, 상담신청내역을 볼 수 있어야 합니다.
- 상담사페이지(관리자화면), 비즈니스 로직처리(백엔드), 상담예약페이지(신청자화면) 총 3개의 어플리케이션으로 구성되어야 합니다.
설계 요구사항
1. 시스템 아키텍처 설계
- 전체 시스템 구조도 (프론트엔드-백엔드-데이터베이스 연결 구조)
- 기본적인 계층 구조 설계 (Controller-Service-Repository 패턴)
2. 동시성 제어 및 트랜잭션
- 동시 예약시도 처리방안
- 기본 트랜잭션 처리 방법
과제1 – 설계과제
3. 데이터베이스
- ERD 설계 (테이블 간 관계 정의)
- 기본 쿼리 최적화 고려사항
4. 보안
- JWT 인증 구조
- 기본 권한 관리 (관리자/사용자 구분)
- 기본 보안 고려사항 (입력값 검증, SQL Injection 방어 등)
5. 개발 환경 및 배포
- Docker 기반 개발 환경 구성 방법
- 배포 시나리오 (AWS/GCP 활용)

과제2 – 구현과제
주제: 상담예약 시스템 구현
과제 설명
- 앞서 작성한 설계를 바탕으로 상담예약 시스템을 구현하세요.
- 모든 데이터는 Database에 저장되어야 하며, 설계한 기본 계층 구조를 반영해야 합니다.
- 기본적인 동시성 처리 방식이 구현에 포함되어야 합니다.
구현 범위
- 상담사(관리자) 페이지
- 상담신청(사용자) 페이지
- 비즈니스로직 처리(백엔드)
- 데이터 저장(Database)
기술 스택
- 화면: ReactJS
- 서버: NestJS
- 데이터베이스: PostgreSQL
과제2 – 구현과제
구현 상세 설명
1. 상담사 페이지
- 상담사 로그인, 로그아웃 기능
- 캘린더를 통한 30분 단위. 상담스케줄 생성/수정/삭제 (스케줄당 최대 3명 예약 가능)
- 상담희망자에게 이메일로 상담신청 링크 발송 (본인 이메일로 발송)
- 상담스케줄에서 상담신청내역 조회 및 상담내역 기록
2. 신청자 페이지
- 이메일로 전달받은 링크 구현
- 접속시 캘린더를 통해 해당일의 스케줄 확인 후 특정 시간대 상담 신청
- 동일시간대의 3명의 상담원이 모두 일정이 예약되어있으면 선택 불가
- 신청완료시 데이터 저장 및 완료 메세지 처리
3. 백엔드
- 설계한 아키텍처 구조 기반으로 구현
- Restful 규격의 API 구현
- 일관적이며 복잡도가 낮은 코드 지향
과제2 – 구현과제
4. 데이터베이스
- 설계한 ERD 기반의 테이블 설계
- 중복된 데이터를 지양하고, 정규화 처리
5. 보안
- JWT인증
- 설계단계에서 제시한 보안전략 반영
